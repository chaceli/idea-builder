## 1. 产品定位和目标用户

### 1.1 产品定位

**IDEA Builder 专利生成器** 是一款基于 AI 技术的智能专利创作平台，旨在让专利申请变得像写文章一样简单。

通过深度整合大语言模型（LLM），平台能够帮助用户从创意构思、专利撰写到查重优化的全流程自动化，大幅降低专利申请的技术门槛和成本。

> **📢 V2.0 重大升级**：全新引入**步骤式专利撰写向导**，按中国专利局标准结构分模块引导用户完成撰写；新增**多人协同编辑**与**版本管理**功能；强化**权利要求智能拆分**与**格式规范化**，输出更专业、更合规的专利申请文件。

### 1.2 核心价值主张

| 价值点 | 描述 | V2.0 升级 |
|--------|------|-----------|
| **零门槛** | 无需专利撰写经验，AI 辅助完成专业文档 | ✨ **步骤式向导**，按专利结构引导 |
| **高效率** | 从想法到完整专利申请材料，分钟级生成 | ⚡ **智能拆分**+**批量处理** |

| **团队协作** | - | 👥 **多人协同**+**版本管理** |

### 2.2 功能详情

#### 2.2.1 模块一：AI 专利创意生成 (Idea Generation)

**功能描述：** 用户输入技术领域关键词，AI 自动分析并生成专利创意想法。

| 功能点 | 详细说明 |
|--------|----------|
| **领域输入** | 用户输入技术领域（如：新能源汽车、智能家居、医疗设备）或直接描述问题 |
| **AI 创意生成** | 基于输入领域，LLM 生成 3-5 个专利创意候选 |
| **创意详情** | 每个创意包含：发明名称、核心技术方案、适用场景、预期效果 |
| **灵感激发** | 提供关联领域热门技术趋势供参考 |
| **收藏/选中** | 用户可收藏创意或直接选中进入撰写流程 |

**AI 模型：**
- 使用 MiniMax API (或可切换至 OpenAI/Claude)
- 模型：MiniMax-M2.5
- Prompt 优化：专利领域专用提示词模板

#### 2.2.2 模块二：专利撰写 (Patent Writing) - V2.0 重大升级

**功能描述：** 基于选中的创意，通过**步骤式向导**分模块引导用户完成专利申请文件撰写。

> V2.0 引入全新的**步骤式撰写向导**，严格按照中国专利局标准结构，引导用户逐步完成：技术领域 → 背景技术 → 发明内容 → 附图说明 → 具体实施方式 → 权利要求书 → 摘要。每个模块支持 AI 辅助生成与手动编辑，确保内容完整、逻辑严谨。

| 功能点 | 详细说明 | V2.0 升级 |
|--------|----------|----------|
| **专利类型选择** | 发明专利 / 实用新型专利 / 外观设计专利 | - |
| **发明名称** | AI 推荐 + 用户自定义 | ✨ 支持多次智能推荐 |
| **步骤式向导** | - | **新增**：5步引导流程 |
| **分模块填写** | - | **新增**：按结构分步填写 |
| **实时预览** | - | **新增**：侧边栏实时预览 |
| **自动保存草稿** | - | **新增**：每30秒自动保存 |

|------|----------|------|-------------|

| **引用关系校验** | 检查权利要求之间的引用关系是否正确 |
| **保护范围评估** | 评估权利要求的保护范围是否合理 |
| **法律用语校验** | 检查是否使用规范的法律术语 |

| **改进建议** | 提供规避相似专利的修改方向 | - |
| **引用建议** | 推荐引用的对比文件 | - |
| **查重报告导出** | - | **新增**：生成查重报告 |

**技术实现：**

#### 5.1.2 专利 (Patent)

```typescript
interface Patent {
  id: string;                     // 专利唯一标识 (UUID)
  userId: string;                 // 所属用户 ID
  
  // 基本信息
  title: string;                  // 发明名称
  type: 'invention' | 'utility' | 'design'; // 专利类型
  status: 'draft' | 'writing' | 'completed' | 'archived'; // 状态
  
  // 撰写进度 (V2.0 新增)
  writingProgress?: {
    basic: boolean;               // 基本信息
    technicalField: boolean;      // 技术领域
    background: boolean;         // 背景技术
    inventionContent: boolean;    // 发明内容
    drawingsDescription: boolean; // 附图说明
    implementation: boolean;      // 具体实施方式
    claims: boolean;              // 权利要求书
    abstract: boolean;            // 摘要
  };
  
  // 创意信息
  ideaDescription?: string;      // 原始创意描述
  ideaGenerated?: string;         // AI 生成的创意详情
  
  // 专利内容 (JSON 存储)
  content: {
    technicalField: string;      // 技术领域
    background: string;          // 背景技术
    inventionContent: string;    // 发明内容
    drawingsDescription: string; // 附图说明
    implementation: string;      // 具体实施方式
    claims: Claim[];              // 权利要求书
    abstract: string;             // 摘要
  };
  
  // 查重结果
  duplicateCheck?: {
    similarity: number;          // 相似度 0-100
    riskLevel: 'low' | 'medium' | 'high';
    suggestions: string[];
    checkedAt: Date;
  };
  
  // 优化结果
  optimization?: {
    qualityScore: number;         // 质量评分 0-100
    issues: string[];             // 发现的问题
    suggestions: string[];        // 优化建议
    optimizedAt: Date;
  };
  
  // 版本信息 (V2.0 新增)
  versions?: PatentVersion[];
  currentVersion?: string;       // 当前版本ID
  
  // 协作信息 (V2.0 新增)
  collaboration?: {
    enabled: boolean;
    members: CollaborationMember[];
  };
  
  createdAt: Date;
  updatedAt: Date;
}

interface Claim {
  id: number;                     // 权利要求编号
  text: string;                    // 权利要求内容
  dependentOn?: number;           // 依赖的权利要求编号
  isIndependent: boolean;         // 是否为独立权利要求 (V2.0 新增)
}

// V2.0 新增：版本信息
interface PatentVersion {
  id: string;
  versionNumber: number;          // 版本号 v1, v2, v3...
  content: Patent['content'];     // 该版本的完整内容
  createdBy: string;              // 创建者
  createdAt: Date;
  description?: string;           // 版本描述
  isAutoSave: boolean;            // 是否为自动保存
}

// V2.0 新增：协作成员
interface CollaborationMember {
  userId: string;
  role: 'owner' | 'editor' | 'viewer';
  assignedModules?: string[];     // 分配的模块
  joinedAt: Date;
}
```

### 5.2 数据库表结构 (MySQL)

```sql
-- 用户表
CREATE TABLE users (
  id VARCHAR(36) PRIMARY KEY,
  email VARCHAR(255) UNIQUE,
  api_provider ENUM('minimax', 'openai', 'claude') DEFAULT 'minimax',
  api_key TEXT NOT NULL,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);

-- 专利表 (V2.0 新增字段: writing_progress, versions, collaboration)
CREATE TABLE patents (
  id VARCHAR(36) PRIMARY KEY,
  user_id VARCHAR(36) NOT NULL,
  title VARCHAR(500) NOT NULL,
  type ENUM('invention', 'utility', 'design') NOT NULL,
  status ENUM('draft', 'writing', 'completed', 'archived') DEFAULT 'draft',
  
  idea_description TEXT,
  idea_generated TEXT,
  
  content JSON,
  duplicate_check JSON,
  optimization JSON,
  
  -- V2.0 新增字段
  writing_progress JSON,           -- 撰写进度
  current_version VARCHAR(36),    -- 当前版本ID
  
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  
  FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE
);

-- V2.0 新增：专利版本表
CREATE TABLE patent_versions (
  id VARCHAR(36) PRIMARY KEY,
  patent_id VARCHAR(36) NOT NULL,
  version_number INT NOT NULL,
  content JSON NOT NULL,
  description VARCHAR(500),
  is_auto_save BOOLEAN DEFAULT FALSE,
  created_by VARCHAR(36),
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  
  FOREIGN KEY (patent_id) REFERENCES patents(id) ON DELETE CASCADE
);

-- V2.0 新增：协作成员表
CREATE TABLE collaboration_members (
  id VARCHAR(36) PRIMARY KEY,
  patent_id VARCHAR(36) NOT NULL,
  user_id VARCHAR(36) NOT NULL,
  role ENUM('owner', 'editor', 'viewer') DEFAULT 'viewer',
  assigned_modules JSON,          -- 分配的模块列表
  joined_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  
  FOREIGN KEY (patent_id) REFERENCES patents(id) ON DELETE CASCADE,
  FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE,
  UNIQUE KEY unique_patent_user (patent_id, user_id)
);

-- 创意表
CREATE TABLE ideas (
  id VARCHAR(36) PRIMARY KEY,
  user_id VARCHAR(36) NOT NULL,
  patent_id VARCHAR(36),
  
  input_domain VARCHAR(500) NOT NULL,
  input_description TEXT,
  output_ideas JSON,
  
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  
  FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE,
  FOREIGN KEY (patent_id) REFERENCES patents(id) ON DELETE SET NULL
);

-- 索引
CREATE INDEX idx_patents_user_id ON patents(user_id);
CREATE INDEX idx_patents_status ON patents(status);
CREATE INDEX idx_ideas_user_id ON ideas(user_id);
CREATE INDEX idx_patent_versions_patent_id ON patent_versions(patent_id);
CREATE INDEX idx_collaboration_patent_id ON collaboration_members(patent_id);
```

### 5.3 Redis 缓存设计

| Key 模式 | 类型 | 说明 | TTL |
|----------|------|------|-----|
| `user:{id}:session` | String | 用户会话 token | 7 天 |
| `patent:{id}:lock` | String | 专利编辑锁（防止并发） | 5 分钟 |
| `patent:{id}:autosave` | String | 自动保存缓存 | 1 分钟 |
| `api:{userId}:quota` | Hash | API 调用配额 | 每日重置 |
| `cache:idea:{hash}` | String | 创意生成缓存（相同输入） | 1 小时 |
| `cache:claims:{patentId}` | String | 权利要求拆分结果缓存 | 24 小时 |
| `cache:format:{patentId}` | String | 格式检测结果缓存 | 1 小时 |

**V2.0 新增：智能权利要求拆分 Prompt**
```
你是一位专利审查专家。请根据以下发明内容，智能拆分权利要求书：

发明名称：{title}
技术方案：{technicalSolution}

请按以下要求拆分权利要求：
1. 识别并生成1-2条独立权利要求（覆盖核心技术特征）
2. 生成3-5条从属权利要求（对独立权利要求进行限定）

输出格式：
- 独立权利要求 X（编号）：[权利要求内容]
- 从属权利要求 Y（引用X）：[权利要求内容]

注意：
- 独立权利要求应具有最宽泛的保护范围
- 从属权利要求应从结构、方法、参数等角度进行限定
- 引用关系必须正确，不能循环引用
```
