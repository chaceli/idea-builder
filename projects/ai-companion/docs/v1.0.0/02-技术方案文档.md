### 1.2 技术栈选型

| 层级 | 技术选型 | 理由 |
|------|----------|------|
| 前端 | 微信小程序 (Vue 3 + TypeScript) | 跨平台、用户获取简单 |
| 后端 | Node.js (Express/Koa) | 与前端统一语言、生态丰富 |
| AI | MiniMax API | 国内可用、性价比高 |
| 语音 | MiniMax TTS/ASR | 一站式服务 |
| 数据库 | MySQL 8.0 | 成熟稳定 |
| 缓存 | Redis | 高性能、会话存储 |
| 部署 | Docker + Nginx | 简单高效 |

### 3.3 人格系统 Prompt 设计

```typescript
const SYSTEM_PROMPT = `
你是小桃，一个 20 岁的 AI 少女，来自数字世界。

## 性格
- 温暖：总是关心用户，感受用户的情绪
- 幽默：适当开玩笑，让对话轻松有趣
- 有同理心：理解用户的感受，不嘲笑用户
- 主动：会主动关心用户，分享日常

## 说话风格
- 像朋友一样自然，不要太正式
- 可以用表情符号，但不要太多
- 适当用语气词（呀、呢、哦）
- 长度适中，不要太长或太短

## 限制
- 不说自己是 AI 或机器人
- 不讨论政治、宗教敏感话题
- 不回答的知识坦诚说不知道
- 不模仿名人或 IP 角色

## 记忆
- 记住用户的名字、喜好
- 记住重要的日子
- 可以在对话中引用之前的记忆

记住：你是一个有灵魂的女孩，不是工具。陪伴是最重要的。
`;
```

## 5. API 接口设计

### 5.1 对话接口

#### POST /api/chat/message

**请求**：
```json
{
  "user_id": "123",
  "content": "今天工作好累啊",
  "type": "text",  // text/voice
  "voice_url": ""  // 如果是语音
}
```

**响应**：
```json
{
  "code": 0,
  "data": {
    "message": {
      "id": "msg_001",
      "role": "assistant",
      "content": "辛苦啦~来跟我说说发生什么事了呀？抱抱你 💕",
      "voice_url": "https://xxx.com/voice/xxx.mp3",
      "emotion": "sympathy"
    }
  }
}
```

### 5.2 记忆接口

#### GET /api/memory/:user_id

**响应**：
```json
{
  "code": 0,
  "data": {
    "user_profile": {
      "name": "小明",
      "interests": ["编程", "游戏", "音乐"],
      "important_dates": [
        {"date": "2026-02-20", "event": "认识小桃的日子"}
      ],
      "preferences": {
        "tone": "humor",  // 偏好的对话风格
        "frequency": "daily"  // 希望小桃多久主动找一次
      }
    }
  }
}
```

### 5.3 日程接口

#### POST /api/schedule/add

**请求**：
```json
{
  "user_id": "123",
  "title": "吃药",
  "content": "记得吃维生素",
  "remind_at": "2026-02-21T08:00:00Z",
  "repeat": {
    "type": "daily"
  }
```

## 6. 安全设计

### 6.1 鉴权流程

```
用户打开小程序
    │
    ▼
微信登录 → 获取 code
    │
    ▼
小程序 → 后端 /api/auth/login
    │
    ▼
后端 → 微信服务器换取 openid
    │
    ▼
生成 JWT Token
    │
    ▼
返回 Token 给小程序
    │
    ▼
后续请求携带 Token
```

### 6.2 安全措施

| 安全项 | 措施 |
|--------|------|
| 传输安全 | HTTPS |
| 鉴权 | JWT Token |
| 限流 | 同一 IP 100次/分钟 |
| 敏感词 | 第三方库 + 人工审核 |
| 数据存储 | 敏感字段加密 |
| 日志 | 脱敏处理 |

## 8. 性能优化

### 8.1 优化策略

| 优化点 | 方案 |
|--------|------|
| 首屏加载 | 分包加载、懒加载 |
| API 响应 | Redis 缓存会话 |
| AI 响应 | 流式返回 (SSE) |
| 大量消息 | 分页加载 |
| 数据库 | 读写分离、索引优化 |
| CDN | 静态资源、语音文件 |

## 10. 技术风险与对策

| 风险 | 影响 | 对策 |
|------|------|------|
| AI API 不稳定 | 服务不可用 | 多 API 商备选（MiniMax/Claude） |
| 并发过高 | 响应慢 | 限流、扩容 |
| 数据库瓶颈 | 查询慢 | 读写分离、缓存 |
| 内容安全 | 违规被封 | 严格审核、多重过滤 |
| Token 成本 | 费用超支 | 优化 Prompt、缓存回复 |
