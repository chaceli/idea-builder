# IDEA Builder 技术方案

## 1. 技术栈

| 层级 | 技术 |
|------|------|
| 前端 | Vue 3 + TypeScript + Vite |
| 状态管理 | Pinia |
| 路由 | Vue Router |
| UI 组件 | Element Plus |
| 后端 | Node.js + Express + Koa |
| 数据库 | MySQL + Redis |
| AI | MiniMax API |

## 2. 系统架构

```
┌─────────────────┐
│   前端 (Vue3)   │
└────────┬────────┘
         │ HTTP/WebSocket
┌────────▼────────┐
│  后端 (Node.js) │
└────────┬────────┘
         │
    ┌────┴────┐
    │         │
┌───▼──┐  ┌──▼────┐
│MySQL │  │Redis  │
└──────┘  └───────┘
         │
    ┌────▼────┐
    │MiniMax  │
    │  API    │
    └─────────┘
```

## 3. API 设计

### 3.1 提示词生成
- `POST /api/prompt/generate` - 生成提示词
- `POST /api/prompt/optimize` - 优化提示词
- `GET /api/prompt/templates` - 获取模板列表

### 3.2 专利生成
- `POST /api/patent/generate` - 生成专利
- `GET /api/patent/list` - 专利列表
- `GET /api/patent/:id` - 专利详情
- `PUT /api/patent/:id` - 更新专利

### 3.3 创意工具
- `POST /api/tool/script` - 短视频脚本
- `POST /api/tool/bp` - 商业计划书

## 4. 数据库设计

### 4.1 用户表 (users)
- id, openid, nickname, avatar, created_at

### 4.2 提示词表 (prompts)
- id, user_id, title, content, type, created_at

### 4.3 专利表 (patents)
- id, user_id, title, content, type, status, created_at

### 4.4 创意内容表 (contents)
- id, user_id, type, title, content, created_at

## 5. 安全考虑

- API Key 加密存储
- 输入内容敏感词过滤
- 请求频率限制
